# テスト仕様レポート（概要）

本レポートは、本プロジェクトの主要仕様を自動テストで検証する範囲を日本語で要約したものです。詳細は `build/reports/tests/test/index.html` を参照してください。

## 対象バージョン
- JDK 25
- Gradle Wrapper 9.0（JUnit 5 使用）

## テスト項目
- Move（3すくみの相性）
  - 斬>打、打>突、突>斬、同手は相性なし
  - 3×3全組み合わせで `Outcome` が期待通りか検証
- BattleEngine（命中/ダメージ仕様）
  - 命中率: 基本70% + 技×2%。相性＋/－で±20pp、5〜95%クランプ
  - 端挙動: 異常値（負の技）でも5%にクランプされることを確認
  - ダメージ: 10 + 力×3 − 守備×2。相性±で±5、最小1を保証
- Gauntlet（5人抜き進行）
  - 5体の敵が順に生成され、合計ポイント 12/13/14/15/16 を満たす
  - 各ステータスは 0〜10 の範囲

## 除外（現時点）
- UI（Swing）の見た目・入力制御の自動テスト
  - 主要ロジックはモデル層で検証
  - 将来、UIテストを導入する場合はAssertJ-Swing等を検討

## 実行方法
- `./gradlew test`
- レポート: `build/reports/tests/test/index.html`

---
テストは仕様のドキュメント化を兼ねています。新規仕様追加時は、テストの追加/更新も同時に行ってください。
